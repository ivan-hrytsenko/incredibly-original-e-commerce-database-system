# Incredibly Original E-Commerce Database System

**Incredibly Original E-Commerce Database System** — це унікальна інформаційна система для управління сучасним маркетплейсом. Проєкт зосереджений на створенні чистої архітектури даних, що дозволяє легко масштабувати каталог товарів та підтримувати складні зв'язки між замовленнями й користувачами.

---

## Вимоги

### Призначення системи
Система автоматизує процеси роздрібної онлайн-торгівлі, забезпечуючи точний облік товарних запасів, керування клієнтськими профілями та безпечне зберігання історії транзакцій.

### Функціональні вимоги
* **Гнучкий каталог:** Підтримка зв'язків "багато-до-багатьох" між товарами та категоріями.
* **Багатоваріантність адрес:** Можливість збереження кількох локацій доставки для одного клієнта (дім, офіс, поштомат).
* **Історичність даних:** Фіксація ціни товару безпосередньо в замовленні на момент його створення для захисту від змін у майбутньому.

### Бізнес-правила
1. **Цілісність кошика:** Замовлення не може бути порожнім; воно повинно містити принаймні одну позицію (Order Item).
2. **Логістика:** Кожне замовлення жорстко прив'язується до однієї адреси зі списку адрес клієнта.
3. **Унікальність:** Реєстрація клієнта можлива лише за унікальною адресою електронної пошти.
4. **Статистика:** Кожен товар може належати до необмеженої кількості категорій для покращення SEO та навігації.

## Сутності та атрибути

| Сутність | Основні атрибути (PK - первинний ключ) | Опис |
| :--- | :--- | :--- |
| **Customer** | **customer_id** (PK), first_name, last_name, email, phone | Профіль покупця |
| **Address** | **address_id** (PK), city, street, house, zip_code, *customer_id* | Точки доставки |
| **Product** | **product_id** (PK), name, description, price, stock_qty | Картка товару |
| **Category** | **category_id** (PK), title, description | Групування товарів |
| **Order** | **order_id** (PK), order_date, status, total, *customer_id*, *address_id* | Документ замовлення |

### Асоціативні сутності (M:N)
* **Order_Item**: Поєднує `Order` та `Product`. Зберігає `quantity` (кількість) та `unit_price` (фіксована ціна на момент покупки).
* **Product_Category**: Поєднує `Product` та `Category` для реалізації гнучкого каталогу.

---

## Зв'язки

* **Customer 1 : N Address**: Користувач може мати профіль з багатьма адресами.
* **Customer 1 : N Order**: Один клієнт може генерувати велику кількість замовлень.
* **Order M : N Product**: Реалізовано через проміжну таблицю `Order_Item`.
* **Product M : N Category**: Реалізовано через проміжну таблицю `Product_Category`.
* **Address 1 : N Order**: Одна адреса може бути використана для багатьох замовлень, але замовлення завжди їде тільки на одну адресу.

---

## Припущення та обмеження
* Видалення клієнта призводить до каскадного видалення його адрес, але замовлення залишаються в системі з поміткою "Deleted User" для збереження фінансової звітності.
* Система підтримує лише одну валюту для розрахунків (на даному етапі).
* Мінімальна кількість товару для замовлення — 1 одиниця.
